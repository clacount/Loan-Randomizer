Rules used by this app
  • The number of loan officers can be any size.
  • Each loan is assigned to one random officer.
  • Built-in and custom seasonal loan types are supported.
  • Custom loan types can be activated by date range or added manually at any time.
  • Only active loan types are available for new assignments.
  • Each loan should include an Amount Requested unless that loan type is configured to skip it.
  • Each active loan type is shuffled and balanced separately.
  • Within each type, the app compares a balanced fairness score using type mix, goal dollars, and loan count, then picks the officer that produces the most even team result.
  • A CSV history file is kept in the selected output folder so the next run can continue balancing from prior history.
  • Loan names and IDs are tracked across sessions to prevent duplicates.
