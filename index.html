Rules used by this app
  • The number of loan officers can be any size.
  • Each loan is assigned to one random officer.
  • Each loan must be marked as Auto, Personal, Credit Card, or Collateralized.
  • Each loan should include an Amount Requested so dollar totals can be balanced fairly.
  • Credit Card loans are tracked by unit count and type, but their dollar amounts do not count toward goal-dollar balancing.
  • Each loan type is shuffled and balanced separately.
  • Within each type, the app compares a balanced fairness score using type mix, goal dollars, and loan count, then picks the officer that produces the most even team result.
  • A CSV history file is kept in the selected output folder so the next run can continue balancing from prior dollar and type history.
  • Loan officers missing from a single run stay in the CSV history so their totals return the next time the folder is loaded.
  • Fairness is measured against each officer's active sessions, so time off does not force a catch-up spike when someone returns.
